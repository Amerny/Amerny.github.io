寄存器

8086CPU所有寄存器都是16位的

通用寄存器：ax,bx,cx,dx 。都可以分为ah和al，各8位

cx 循环结构的循环次数记录

段寄存器：cs,ds,es,ss。8086CPU不支持数据直接送入段寄存器

cs ：代码段寄存器

ds：要访问的数据段的段地址，读写一个内存单元

ss：栈顶的段地址，

si和di和bx的功能相近，不能够分成两个8位寄存器

bp寄存器

标志寄存器flag![截屏2021-09-08 下午9.30.50](汇编小结/截屏2021-09-08 下午9.30.50.png)

ZF：零标志位，记录相关指令执行之后的结果，zf=1，结果为0；zf=0，结果不为0

PF：奇偶标志位，记录相关指令执行之后的结果，pf=1，结果所有位中1的个数为偶 数；pf=0，1的个数为奇数

SF：符号标志位，记录相关指令执行之后的结果，sf=1，结果为负；sf=0，结果非负

CF：进位标志位，记录无符号数运算的时候，记录最高有效位向更高位的进位值或者借 位值，有进位或者借位，cf=1；没有则cf=0

OF：溢出标志位，记录有符号数运算的结果是否发生了溢出，如果发生溢出of=1；

DF：方向标志位，串处理指令中，控制每次操作后si，di的增减，df=0，si和di增；

![截屏2021-09-09 上午10.33.35](汇编小结/截屏2021-09-09 上午10.33.35.png)

8086CPU任意时刻，将CS：IP指向的内容当做指令进行——CS*16+IP的单元开始执行，然后IP值增加当前指令的长度。简要执行过程如下

  1. 从CS：IP指向的内存单元读取指令，读取的指令进入指令缓冲器
  2. IP=IP+所读指令的长度，从而指向下一条指令
  3. 执行指令，回到1

Debug功能的常用命令汇总

  • R 命令查看、改变CPU的寄存器的内容。用法r或者r ax
  • D命令查看内存中的内容。用法d 段地址（段寄存器）：偏移地址，列出128个内存单元内容
  • E命令改写内存中的内容。e 起始地址 数据 数据
  • U命令将机器指令翻译成汇编指令。用法u 段地址：偏移地址
  • T命令执行一条机器指令。t执行当前CS：IP指向的命令
  • A以汇编指令的格式在内存中写入一条机器指令。用法 a 段地址：偏移地址

8086CPU任意时刻,SS:SP指向栈顶元素

push ax的过程

  1. sp=sp-2, ss:sp指向栈顶前面的单元，以当前栈的,以当前栈顶前面的单元为新的栈顶；
  2. 将ax中的内容送入SS：SP的内存单元处，SS：SP此时指向新栈顶。

![截屏2021-09-07 下午5.23.17](汇编小结/截屏2021-09-07 下午5.23.17.png)

pop ax的过程和push相反。

在8086中，只有bx，si，di，bp这四个寄存器可以用在[ ]..中进行内存单元的寻址，并且只能以单个出现，或者4种组合：bx和di，bx和si，bp和di，bp和si并且都可以加上常数或者立即数idata，但凡包含bp，段地址都在ss中，其他组合的段地址在ds中。

常见指令

mov ax,[bx] ：bx中存放的数据作为一个偏移地址EA，段地址SA默认在ds中，将SA：EA 的数据送入ax。

mov [bx],ax：原理同上，ax中的数据送入内存SA：EA中。

mov ax,[bx+200]：ds：200+bx中的数值或者也可以写成 mov ax,200[bx];mov ax,[bx].200 。

mov ax,[bx+si]：ax=ds*16+bx的值+si的值，也可以写成mov ax,[bx][si]。

mov ax,[bx+si+200]：ax=ds*16+bx值+si值+200，也可以写成mov ax,200[bx][si]/mov [bx].200[si]/mov [bx][si].200。

inc bx：bx中的内容加一。

loop s：cx=cx-1 -> 判断cx的值，不为0则转至标号s的地址处执行。

汇编源程序中，数据不能以字母开头，可以前面加0代替

debug中的mov ax,[0]表示把ds：0处的数据送入ax；masm中的mov ax,[0]表示把常量0送入ax,解决方法是在 []的前面显示给出段地址的段寄存器。

描述性符号reg和sreg，reg表示一个寄存器；sreg表示一个段寄存器，包括ds,ss,cs,es

字长的处理

可以处理byte和word两种数据，1个word=2个byte

  • 通过寄存器指明要处理的尺寸

    ax，bx，ds，表示字操作；al，bl表示字节操作

  • 没有寄存器指明的情况下，用 word ptr或者 byte ptr指明内存单元的长度。

    比如move byte ptr ds:[0],1

  • 其他方法，比如push 指令只进行字操作。

一般来说，可以用[bx+idata+si]访问结构体中的数据，用bx定位整个结构体，idata定位某一个数据项，si定位数组项的每个元素C语言中的student.name[i]=汇编中的bx.10h[si]。

div指令和mul指令

  • dive指令

除数有8位和16位，在一个reg或者内存单元中；被除数默认在ax或者ax和dx中；

如果除数为8位，则被除数为16位，默认在ax中存放，al存储商，ah存储余数；如果除数为16位，则被除数为32位，在ax和dx中，dx放高16位，ax存储商，dx存储余数。

比如100001/100

  • mul指令

    两个相乘的数如果都是8位，一个默认在AL中，另一个在8位reg或内存单元中，结果默认在AX中；如果都是16位，一个默认在AX中，另一个在16位reg或内存单元中，结果高位在DX中，低位在AX中。

    比如100*10000

dd指令和dup指令

db定义字节型数据

dw定义字型数据

dd定义双字型数据

db 3 dup (0)表示定义了3个字节，值都是0，格式是db 重复的次数 dup （重复的数据）

对于db 字符数据，相当于定义每一个字母的占位为8位，比如 ‘hello’，就代表‘h’ ，’e’…

adc指令和sbb指令

adc是带进位加法指令 ====> adc 操作对象1 ，操作对象2 ==>对象1=对象1+对象2+CF

sbb是带借位减法指令====> sbb 操作对象1 ，操作对象2 ==>对象1=对象1-对象2-CF

操作符offset

offset 标号 ：取得标号的偏移地址

jmp指令

  • jmp short 标号（转到标号处执行命令）

    段内短转移，对IP的修改范围是-128 到127

    CPU在执行jmp指令的时候并不需要转移的目的地址，而包含的是转移的位移，这个位移是编译器根据标号计算出来的。最终实现的功能是IP=IP+8位位移（补码表示）。

  • jmp near ptr 标号

    段内近转移，对IP的修改范围是-32768 - 32767，IP=IP+16位位移（补码表示）。

  • jmp far ptr 标号

    段间转移（远转移）

    cs=标号所在段的段地址；IP=标号在段中的偏移地址。

  • jmp word ptr 内存单元地址（段内转移）

  • jmp dword ptr 内存单元地址（段间转移）

    从内存单元地址处开始存放着两个字，高地址处的字是转移的目的段地址（CS），低地址处是转移的目的偏移地址（IP)

  • jcxz 标号（有条件转移指令）

    如果cx=0，转到标号处执行。

所有有条件转移指令和循环指令都是短转移，对IP的修改为8位

call 和ret指令

ret指令修改IP的内容，进转移；retf指令修改CS和IP的内容实现远转移。

执行ret指令相当于进行 pop IP；retf指令相当于进行 pop IP；pop CS；

call指令时，将当前IP和CS压入栈中，并转移至标号处。

执行call指令相当于进行 push IP；jmp near ptr 标号;

call far ptr 标号 相当于进行 push CS; push IP; jmp far ptr 标号

cmp指令

格式：cmp 操作对象1，操作对象2

功能：计算对象1-对象2，不保存结果，仅仅对标志寄存器进行设置

cmp ax,bx

  1. ax=bx,ax-bx=0, zf=1
  2. ax!=bx,zf=0
  3. ax<bx,产生借位，cf=1
  4. ax>=bx,cf=0
  5. ax<=bx,cf=1,zf=1
  6. ax>bx,cf=0,zf=0

串传送指令

movsb 功能是将ds：si指向的内存单元中的字节送入es：di中，然后根据df的值，将di和si递增或者递减。

movsw 功能是将ds：si指向的内存单元中的字送入es：di中，然后根据df的值，将di和si递增2或者递减2。

一般来说，这两个指令都配合rep使用

rep movesb相当于s:movesb loop s，根据cx的值重复执行后面的传送指令。

对方向标志位df进行设置 cld指令：将df置0； std指令：将df置1

pushf和popf指令

pushf功能是将标志寄存器的值压入栈中，而popf是从栈中弹出数据送入标志寄存器中。

中断

内中断

cpu执行下面情况的时候会发生中断

  1. 除法错误，中断类型码为0
  2. 单步执行，中断类型码为1
  3. 执行into指令，中断类型码为4
  4. 执行int指令，指令格式为int n，n为中断类型码

中断过程

  1. 取得中断类型码
  2. 标志寄存器值入栈
  3. 设置flag的第八位TF和第九位IF的值为0。TF=1引发单步中断，IF=0，禁止其他的可屏蔽中断。
  4. CS的内容入栈
  5. IP的内容入栈
  6. 从内存地址为中断类型码*4和中断类型码*4+2的两个字单元中读取中断处理程序的入口地址设置IP和CS。IP=N*4,CS=N*4+2

中断处理程序的编写方法

  1. 保存用到的寄存器
  2. 处理中断
  3. 恢复用到的寄存器
  4. 用iret指令返回，相当于执行，pop IP，pop CS， popf。

中断向量表是pc系统中最重要的内存区，用来存放中断处理程序的入口地址，一般情况下，从0000：0200至0000：02ff的256个字节空间所对应的中断向量表是空的

CPU执行指令 mov ss，ax之后，不响应任何中断，包括单步中断。

int 指令

CPU执行int n指令，相当于引发一个n号中断的处理程序，过程如下

  1. 取中断类型码
  2. 标志寄存器入栈，IF=0，TF=0
  3. CS，IP入栈
  4. IP=n*4，CS=n*4+2

BIOS的 int 10h中断例程

ah=2表示引用中断例程的2号子程序，功能为设置光标位置

ah=9表示调用9号子程序，功能为在光标位置显示字符

端口

端口的读写指令只有in和out

16位的CPU，最多可以定位2^16=64KB个不同的端口，范围为0-65535

​ ps: 在in和out指令中，只能使用ax或者al来存放从端口中读入的数据或者要发送的数据，访问8位端口使用al，16位使用ax。

外中断

可屏蔽中断（CPU可以不响应的外中断）

CPU响应与否外中断取决于标志寄存器IF的设置，当检测到CPU的中断信息时，如果IF=1，则执行完当前中断之后响应中断；如果IF=0，不响应。

不可屏蔽中断

对于8086，不可屏蔽中断的中断类型码固定为2，所以中断过程不需要取中断类型码。
